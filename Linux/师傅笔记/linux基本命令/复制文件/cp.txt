cp，复制文件、文件夹。

一、复制文件

语法：cp 源文件 路径/目标文件

1.复制文件到当前目录里不需输入路径。如:

例1：cp data.txt backup.txt ，表示复制文件data.txt到当前目录里并改名为backup.txt。这两个文件内容是一样的。

2.复制文件到其它目录里时，如果不更改文件名不需要输入目标文件名称，语法：cp 源文件 路径/ ,如：

例2：cp data.txt /home/backup/ ，表示复制data.txt到/home/backup目录下。

"cp data.txt /home/backup/" = "cp data.txt /home/backup/data.txt" ,因为不更改文件名，所以目标文件名称data.txt可忽略。

注意：目标目录/home/backup/必须是存在的。

二、复制文件夹

语法：cp -r 源文件夹 路径/目标文件夹名称

1.与复制文件一样，复制文件夹到当前目录里不需输入路径。

例3：cp -r data backup ，表示复制文件夹data到当前目录下，并改名为backup。这两个文件夹里的内容是一样的。

注意：目标文件夹backup是不存在的；要是存在，则表示复制文件夹data到当前目录下的backup子目录里。表示的意义不一样。

2.复制文件夹到其它目录里时，要是不更改文件夹名称不需要输入目标文件夹名称。

例4：cp -r data /backup ，表示复制文件夹data到目录/backup下，这时/backup下就有了data文件夹。

"cp -r data /backup" = "cp -r data /backup/data" ，因为不更改所复制文件夹名称，所以目标文件夹名称data可忽略。

注意：目录/backup必须是存在的；如果不存在，则表示复制文件夹data到/下。表示的意义就不一样了。

常用的参数还有：

-v ----显示详细信息

如：cp -rv data /backup

三、如何使cp命令不提示覆盖文件?

如果希望将文件file拷贝到目录dir下，可以执行：cp file dir

但如果dir下已经存在一个名为file的文件的时候，系统总是会提示是否覆盖file。这是一个很好的功能，它能够防止我们由于疏忽对系统造成的损害，但如果不希望看到这些交互的提示信息呢？比如我们需要写一个脚本，将某些文件复制到指定的目录下，这时我们肯定不希望看到提示。可以试一下"-f"选项，"-f"一般表示强制执行（force）。

cp -f file dir

但好像结果并不像我们希望的那样，系统依然会有覆盖提示。为什么呢？

原因很简单，系统将cp命令alias为"cp -i"。当我们执行cp命令的时候，系统往往执行的是"cp -i"，"-i"选项表示有交互的提示信息，所以执行"cp -f"的时候，系统实际执行的是"cp -i -f"， 所以仍然会有覆盖提示。

想要执行原始的cp也很简单，可以在调用cp的时候加入绝对路径，当然还可以直接执行下面的语句：\cp -f file dir
